deconv_peaks {Daisy} R Documentation

Perform peak detection and alignment for LC-MS/GC-MS data

Description

Usage

deconv_peaks(rawfiles,
    workdir = './',
    max.rtwin = 15,
    docker = NULL,
    n.threads = 8,
    call.xcms = TRUE);

Arguments

rawfiles

Character vector of raw data files (mzXML or mzML format).

workdir

Output directory for results (default: "./").

max.rtwin

Maximum retention time window (seconds) for peak alignment (default: 15).

docker

Docker image ID for XCMS execution environment (optional).

n.threads

Number of threads for parallel processing (default: 8).

call.xcms

Logical indicating whether to run XCMS peak detection (default: TRUE). If FALSE, only alignment is performed using existing XCMS output.

Details

The processing pipeline involves two main steps: \enumerate{ \item \strong{XCMS Peak Detection:} When \code{callxcms = TRUE}, runs \code{xcmsfindPeaks()} in parallel batches using matchedFilter algorithm. \item \strong{Peak Alignment:} Aligns peaks across samples using specified RT window and outputs consolidated results. }

Authors

xieguigang@metabolomics.ac.cn

Value

Invisibly returns NULL. Outputs the following files in \code{workdir}: \itemize{ \item \strong{peaktable.csv} - Aligned peak feature table \item \strong{rtshifts.csv} - Retention time shift corrections \item \strong{peakmeta.csv} - Peak metadata (mz, RT, intensity, etc.) \item \strong{rtshifts.pdf} - Visualization of retention time shifts \item \strong{peakset.pdf} - Peak distribution visualization }
